# v0.1.26 Release Notes Draft

Status: draft. Publish this after npm automation is fixed in [#11](https://github.com/EShener/agent-ready/issues/11).

Publishing checklist: [docs/npm-publishing.md](../npm-publishing.md).

## Summary

`agent-ready` v0.1.26 expands detector coverage across frontend, backend, containers, and enterprise stacks so generated agent instructions are more accurate on real repositories.

## Highlights

- Next.js detection now recognizes `next.config.*`, App Router files, and Pages Router files.
- Astro detection now recognizes `astro.config.*` and `.astro` source files.
- Docker Compose detection adds local service guidance to generated `AGENTS.md`.
- SvelteKit and Nuxt are detected from framework configs, dependencies, and route conventions.
- Rails and Laravel detection now infer useful backend commands.
- Spring Boot detection supports Maven and Gradle wrapper command inference.
- .NET and C# projects are detected from `*.csproj`, `*.sln`, `global.json`, and `Program.cs`.
- Go Gin and Rust workspace web fixtures protect existing detector behavior.
- GitLab CI and CircleCI workflow files now count as valid CI signals.
- Python projects now prefer Poetry, PDM, or uv commands when those package manager signals are present.
- Detector coverage is now documented in [docs/detectors.md](../detectors.md).
- Real-world snapshots now include more framework examples in [docs/examples/real-world-snapshots.md](../examples/real-world-snapshots.md).

## Upgrade Notes

The CLI remains zero-dependency and local-first.

```bash
npx --yes @eshen_fox_mie/agent-ready improve --preset team --dry-run
```

For a quick detector check:

```bash
npx --yes @eshen_fox_mie/agent-ready scan
npx --yes @eshen_fox_mie/agent-ready examples
```

## Verification Before Publish

Run locally:

```bash
npm run check
npm test
npm view @eshen_fox_mie/agent-ready version
```

After publishing:

```bash
npm view @eshen_fox_mie/agent-ready version
npx --yes @eshen_fox_mie/agent-ready@latest scan --root .
```
